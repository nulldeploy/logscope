## logscope — что это и зачем

CLI-инструмент для анализа логов в терминале. Главная идея: не просто `tail -f`, а умный фильтр с подсветкой, статистикой и живым режимом. Целевой пользователь — DevOps/разработчик, которому нужно быстро понять что происходит в логах без написания grep-цепочек.

---

## MVP — три команды

**1. `logscope parse`**
Читает лог-файл и выводит строки красиво. Уровни (`ERROR`, `WARN`, `INFO`) подсвечиваются цветом. Можно фильтровать по уровню или ключевому слову.

```bash
logscope parse --file app.log
logscope parse --file app.log --level ERROR
logscope parse --file app.log --grep "timeout"
```

**2. `logscope stats`**
Считает статистику по файлу. Топ повторяющихся ошибок, сколько ERROR/WARN/INFO всего, самый активный час.

```bash
logscope stats --file app.log
```

Вывод примерно такой:
```
Total lines : 1 240
ERROR        : 87
WARN         : 203
INFO         : 950

Top errors:
  42x  Connection timeout to db:5432
  18x  Failed to fetch /api/users
   9x  Permission denied: /var/log/app
```

**3. `logscope watch`**
Висит и следит за файлом в реальном времени. Показывает только нужные строки — с фильтром по уровню или слову. Как `tail -f | grep ERROR`, только с цветом и без боли.

```bash
logscope watch --file app.log
logscope watch --file app.log --level ERROR
logscope watch --file app.log --grep "timeout"
```

## Что внутри (модули)

`core/reader.py` — открывает файл, отдаёт строки. Одна задача.

`core/parser.py` — берёт строку лога, пытается вытащить из неё уровень, время, сообщение. Если не получилось — отдаёт строку как есть.

`core/analyzer.py` — принимает список распарсенных строк, считает статистику через `Counter`.

`core/watcher.py` — бесконечный цикл: читает новые байты из файла, фильтрует, печатает.

`utils/colors.py` — просто словарь: `ERROR → красный`, `WARN → жёлтый`, и т.д. Либо подключаешь `rich` и не паришься.

`cli/` — каждая команда в отдельном файле. `main.py` собирает их через `argparse` с subparsers.

---

## Стек

| Что | Зачем |
|---|---|
| `argparse` | CLI и subparsers — стандартная библиотека, ничего устанавливать не надо |
| `rich` | красивый цветной вывод, прогресс-бары, таблицы — одна библиотека закрывает всё |
| `collections.Counter` | подсчёт топ ошибок — встроено в Python |
| `re` | регулярки для парсинга строки лога — встроено |

Никаких баз данных, никаких веб-серверов. Только файлы и терминал.

---

## С чего начать конкретно

1. Создать структуру папок и пустые файлы
2. Написать `core/reader.py` — просто открыть файл и вернуть строки
3. Написать `core/parser.py` — найти в строке `ERROR`/`WARN`/`INFO` через `re`
4. Подключить `parse` команду и убедиться что цвета работают
5. Только потом — `stats` и `watch`

`watch` пишется последним, но именно он будет главной фичей в портфолио — люди любят смотреть как что-то живёт в реальном времени 🎯

---

**Выжимка:**
- Три команды: `parse`, `stats`, `watch` — каждая делает одну вещь хорошо
- Стек минимальный: `argparse` + `rich` + стандартная библиотека
- Архитектура разделена: `core/` — логика, `cli/` — команды, `utils/` — вспомогалки
- Начинать с `reader` → `parser` → `parse` команда → потом остальное
- `watch` — главная фича для демонстрации, делается последней
